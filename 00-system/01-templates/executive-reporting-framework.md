# 경영진 보고 프레임워크

> 기반: Pyramid Principle (McKinsey), SCQA, MECE, So What/Now What

## 1. 피라미드 원칙 (Pyramid Principle)

**출처**: Barbara Minto, McKinsey & Company (1973)
**핵심**: 결론부터 말하고, 근거로 뒷받침

### 피라미드 구조

```
                    ┌─────────────┐
                    │   결론/주장  │  ← 핵심 메시지 (1문장)
                    └─────────────┘
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
    ┌───────────┐   ┌───────────┐   ┌───────────┐
    │  근거 1   │   │  근거 2   │   │  근거 3   │  ← 핵심 근거 (3개)
    └───────────┘   └───────────┘   └───────────┘
          │               │               │
       세부근거        세부근거        세부근거     ← 상세 데이터
```

### 적용 원칙

```
1. 결론 먼저 (Answer First)
   - 바쁜 경영진은 결론부터 원함
   - "매출이 15% 증가했습니다" (O)
   - "여러 요인을 분석한 결과..." (X)

2. 근거는 MECE하게
   - Mutually Exclusive: 중복 없이
   - Collectively Exhaustive: 누락 없이

3. 3의 법칙
   - 핵심 포인트는 3개로 요약
   - 사람은 3개까지 기억함
```

## 2. SCQA 프레임워크

**출처**: Barbara Minto, McKinsey
**용도**: 스토리라인 구성, 보고서 도입부

### SCQA 구조

| 요소 | 영문 | 설명 | 예시 |
|------|------|------|------|
| S | Situation | 현재 상황 | "2분기 매출 목표는 15억원이었습니다" |
| C | Complication | 문제/변화 | "실제 매출은 12억원으로 목표 대비 20% 미달입니다" |
| Q | Question | 핵심 질문 | "왜 미달했고, 3분기에 어떻게 만회할 것인가?" |
| A | Answer | 해결책/결론 | "3가지 원인이 있으며, 3가지 대응책을 제안합니다" |

### SCQA 작성 예시

```markdown
## 2분기 매출 리뷰

**Situation**: 2분기 매출 목표는 15억원, 전년 대비 20% 성장 계획

**Complication**: 실적 12억원으로 목표 대비 20% 미달, 특히 6월 급감

**Question**: 미달 원인은 무엇이며, 3분기 만회 방안은?

**Answer**:
1. 주요 원인: 대형 고객 이탈(30%), 신규 영업 부진(50%), 계절성(20%)
2. 3분기 대응: 기존 고객 리텐션 강화, 영업 인력 확충, 프로모션 조기 실행
```

## 3. So What / Now What 기법

**출처**: 컨설팅 업계 표준 기법
**핵심**: 데이터 → 의미 → 행동으로 연결

### 3단계 사고

```
Data (데이터)
    ↓ So What? (그래서 뭐?)
Insight (인사이트)
    ↓ Now What? (그래서 어쩌라고?)
Action (액션)
```

### 적용 예시

```
Data: 5월 매출이 4월 대비 15% 감소
    ↓ So What?
Insight: 주력 고객 3곳의 구매 감소가 원인, 경쟁사로 이탈 가능성
    ↓ Now What?
Action: 즉시 해당 고객 미팅 요청, 가격 재협상 검토
```

### 적용 질문

```
"이 분석 결과가 의미하는 바가 뭐야? So What?"
"그래서 우리가 해야 할 액션은 뭐야? Now What?"
"CEO가 이 데이터를 보고 물어볼 질문이 뭘까?"
```

## 4. 숫자 표현 원칙

### 핵심 숫자 선택

```
Rule of 3: 핵심 숫자는 3개 이하
- 기억하기 쉬움
- 집중도 높음
- 의사결정 가능

좋은 예: "매출 15억, 성장률 20%, 신규고객 50개"
나쁜 예: "매출 15억, 원가율 65%, 마진 35%, 성장률 20%, 신규 50개, 이탈 10개..."
```

### 숫자 표현법

| 상황 | 표현 | 예시 |
|------|------|------|
| 큰 숫자 | 단위 변환 | 150,000,000원 → 1.5억원 |
| 비교 | 배수/비율 | 50% 증가 → 1.5배 |
| 맥락 제공 | 벤치마크 | "업계 평균 10% 대비 우리는 15%" |
| 변화 강조 | 차이 | "+5%p" (퍼센트 포인트) |

### 적용 질문

```
"CEO가 기억해야 할 핵심 숫자 3개만 뽑아줘"
"이 숫자가 좋은 건지 나쁜 건지 벤치마크와 비교해줘"
"전분기 대비 가장 큰 변화는 뭐야?"
```

## 5. 슬라이드 구성 템플릿

### 1페이지 요약 (Executive Summary)

```
┌─────────────────────────────────────────────┐
│  제목: [핵심 메시지를 담은 제목]             │
├─────────────────────────────────────────────┤
│  핵심 숫자 (3개)                            │
│  ┌─────┐  ┌─────┐  ┌─────┐                │
│  │ 15억 │  │ +20% │  │ 50개 │                │
│  │ 매출 │  │ 성장 │  │신규  │                │
│  └─────┘  └─────┘  └─────┘                │
├─────────────────────────────────────────────┤
│  주요 인사이트                              │
│  1. [첫 번째 포인트]                        │
│  2. [두 번째 포인트]                        │
│  3. [세 번째 포인트]                        │
├─────────────────────────────────────────────┤
│  권고 사항 / Next Steps                     │
│  • [액션 1]                                 │
│  • [액션 2]                                 │
│  • [액션 3]                                 │
└─────────────────────────────────────────────┘
```

### 슬라이드 제목 작성법

```
나쁜 제목: "2분기 매출 현황"
좋은 제목: "2분기 매출 15% 성장, 목표 달성"

나쁜 제목: "고객 분석"
좋은 제목: "상위 20% 고객이 매출 80% 차지, 집중 관리 필요"

원칙: 제목만 읽어도 메시지 전달
```

## 6. 보고서 체크리스트

### 작성 전 확인

```markdown
- [ ] 청중(CEO/임원)이 알고 싶은 것은?
- [ ] 의사결정에 필요한 정보가 있는가?
- [ ] 시간 제약은? (5분/15분/30분)
```

### 작성 후 검토

```markdown
- [ ] 첫 문장만 읽어도 핵심 메시지 전달?
- [ ] 핵심 숫자가 3개 이하?
- [ ] So What/Now What이 명확?
- [ ] MECE하게 구성됐는가?
- [ ] 용어가 청중 수준에 맞는가?
```

## 7. 실습 적용 예시

### 오피스원 주식회사 종합 보고서

```
Step 1: SCQA 작성
"2분기 전체 데이터를 종합해서 SCQA 형식으로 보고서 도입부를 작성해줘"

Step 2: 핵심 숫자 추출
"CEO가 기억해야 할 핵심 숫자 3개를 뽑아줘.
왜 이 숫자가 중요한지도 설명해줘"

Step 3: So What 도출
"분석 결과에서 가장 중요한 인사이트 3가지는 뭐야?
So What 관점에서 의미를 설명해줘"

Step 4: Now What 액션
"다음 분기에 해야 할 액션 아이템 3가지를 추천해줘.
우선순위와 담당 부서도 제안해줘"

Step 5: 1페이지 요약
"지금까지 분석 내용으로 CEO 보고용 1페이지 요약을 만들어줘.
핵심 숫자, 주요 이슈, 권고 사항 포함해서"
```
