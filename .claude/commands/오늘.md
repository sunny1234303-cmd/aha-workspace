`/오늘` 하나로 하루 기록과 할일 관리를 다 처리합니다. 매일 아침저녁 이 커맨드부터 시작하면 됩니다.

## 기본 동작 (`/오늘`, 인자 없음)

1. **오늘 날짜의 기록 생성/열기**
   - 경로: `./40-schedule/41-daily/YYYY-MM-DD.md`
   - 파일이 없으면 템플릿(`00-system/01-templates/daily-note-template.md`) 기반으로 새로 생성
     - **Google Calendar 연동이 되어 있으면** (`which gcalcli` 확인), 오늘 일정을 가져와 포함:
       ```bash
       gcalcli agenda --tsv
       ```
       TSV 출력을 Markdown 리스트로 변환해 "오늘 일정" 섹션에 추가. 연동 안 되어 있으면 이 섹션은 생략.
     - 변수 치환: `{{date}}`, `{{weekday}}`, `{{yesterday}}`, `{{tomorrow}}`, `{{week}}`
   - 파일이 있으면 현재 내용을 보여줌 (Google Calendar 연동 시 일정 업데이트 여부 확인)

2. **오늘 기록에서 할일 자동 추출**
   - 오늘 기록에 할일로 보이는 문장("~해야 함", "~하기", 체크박스 등)이 있고 아직 `43-todos/active-todos.md`에 없으면, 자동으로 찾아 Inbox 섹션에 추가
   - 알림: "📌 오늘 기록에서 할일 N개를 찾아 추가했어요."

3. **할일 상태 짧게 요약**
   - `40-schedule/43-todos/active-todos.md`를 읽어 Inbox/Today/Overdue 개수만 간단히 보여줌

## 할일만 빠르게 추가 (`/오늘 [내용]`)

기록을 안 열고 할일만 바로 추가하고 싶을 때:

1. 우선순위 태그 파싱: `[urgent]`/`[high]` → high, `[low]` → low, 없으면 normal
2. 프로젝트 태그 파싱: `[프로젝트명]`
3. `40-schedule/43-todos/active-todos.md`의 "📥 Inbox" 섹션에 추가:
   ```markdown
   - [ ] [내용]
     - added: YYYY-MM-DD HH:mm
     - context: [현재 작업 파일/디렉토리]
     - priority: [high/normal/low]
     - project: [있으면]
   ```
4. 확인 메시지: "✅ 할일 추가됨: [내용]"

## 할일 조회 (`/오늘 today` / `/오늘 project` / `/오늘 overdue` / `/오늘 stats`)

- `today` — Today 섹션을 우선순위(high→normal→low)로 정렬해 출력
- `project` — `project:` 필드로 그룹화해서 출력
- `overdue` — `added:` 기준 7일 이상 지난 항목을 오래된 순으로 출력
- `stats` — 전체 개수, 우선순위별/프로젝트별/상태별 통계, 완료율(`completed-todos.md` 대비) 출력

## 완료 처리

`- [x]`로 체크된 항목은 `completed-todos.md`로 옮겨 아카이빙합니다.
