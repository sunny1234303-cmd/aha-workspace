# Commands

사용 가능한 Slash Commands입니다. 실제 쓰는 순서대로 정리했습니다.

## 최초 1회만

- `/setup-workspace` - 워크스페이스 초기 설정 (CLAUDE.md + 데이터베이스 자동 생성). 처음 한 번만 실행하면 됩니다.

## 매일 쓰는 것

- `/오늘` - 오늘 기록 생성/열기 + 할일 캡처·조회를 한 번에

## 생각을 정리하거나 AI를 제대로 부려먹고 싶을 때 (이 순서로)

- `/gather` - 정보 수집 모드 (구조화된 질문으로 먼저 모음. 답변하면 `/reframe`까지 자동으로 이어집니다)
- `/reframe` - 이해 확인 모드 (지금까지 이해한 내용 요약, 맞는지 확인). `/gather` 뒤에는 자동 실행되므로, 그 없이 대화 중간에 확인하고 싶을 때만 따로 부르면 됩니다
- `/truth` - 사실 기반 분석 모드 (객관적으로 분석)

## 대화 중 좋은 생각이 나오면

- `/idea [카테고리]` - 아이디어 추출·저장 (30-knowledge에 쌓임)

## 필요할 때 (보안)

- `/security-check [history]` - 개인정보·시크릿 노출 점검. `git push`할 때마다 자동으로도 걸리지만, 언제든 직접 실행 가능. `history`를 붙이면 커밋 히스토리까지 검사 (Public 전환 전 필수)

---

## Claude Code 기본 내장 커맨드

이 워크스페이스가 만든 게 아니라 Claude Code 자체에 있는 커맨드입니다. 자주 씁니다.

- `/compact` - 대화가 길어졌을 때 요약해서 컨텍스트 정리
- `/clear` - 대화 초기화 (완전히 새로 시작)
- `/resume` - 이전 대화 이어서 하기
- `/help` - 막혔을 때 도움말

---

각 명령어의 자세한 사용법은 해당 파일을 참고하세요.
